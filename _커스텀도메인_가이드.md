# 커스텀 도메인 연결 가이드

현재 배포 URL: `https://dongsoojung.github.io/stargate-ai-gallery/`

원하는 커스텀 도메인(예: `gallery.stargate11.com`)을 이 사이트에 연결하는 절차입니다.

---

## 1. DNS 레코드 설정 (도메인 레지스트라/네임서버 관리 콘솔)

### 서브도메인 (권장) — `gallery.stargate11.com`

`CNAME` 레코드 **1개** 추가:

| Type  | Name      | Value                        | TTL |
|-------|-----------|------------------------------|-----|
| CNAME | `gallery` | `dongsoojung.github.io.`     | 600 |

> 끝에 마침표(`.`) 포함. 일부 DNS 콘솔은 자동 추가되므로 생략해도 무방.

### 루트(Apex) 도메인 — `stargate11.com` 자체에 연결

`A` 레코드 **4개** 추가 (GitHub Pages 공식 IPv4):

| Type | Name | Value            | TTL |
|------|------|------------------|-----|
| A    | `@`  | `185.199.108.153`| 600 |
| A    | `@`  | `185.199.109.153`| 600 |
| A    | `@`  | `185.199.110.153`| 600 |
| A    | `@`  | `185.199.111.153`| 600 |

(선택) IPv6 `AAAA` 레코드:

| Type | Name | Value                           |
|------|------|---------------------------------|
| AAAA | `@`  | `2606:50c0:8000::153`           |
| AAAA | `@`  | `2606:50c0:8001::153`           |
| AAAA | `@`  | `2606:50c0:8002::153`           |
| AAAA | `@`  | `2606:50c0:8003::153`           |

## 2. GitHub Pages 에 도메인 등록

### 방법 A: PowerShell 한 줄 (gh CLI)

```powershell
# 서브도메인 예시
gh api -X PUT repos/DongsooJung/stargate-ai-gallery/pages -f cname='gallery.stargate11.com' -F https_enforced=true
```

### 방법 B: GitHub 웹 UI

1. 저장소 → Settings → Pages
2. Custom domain 입력: `gallery.stargate11.com` → Save
3. DNS 전파 확인 후 `Enforce HTTPS` 체크 (자동 Let's Encrypt 인증서 발급까지 최대 24시간)

위 두 방법 모두 실행 시 저장소 루트에 `CNAME` 파일이 자동 생성/커밋됩니다.

---

## 3. 전파 확인

```powershell
# DNS 전파 확인
nslookup gallery.stargate11.com

# HTTP 응답 확인
curl -I https://gallery.stargate11.com/

# GitHub 측 검증 상태
gh api repos/DongsooJung/stargate-ai-gallery/pages | ConvertFrom-Json | Select-Object html_url, cname, status, protected_domain_state
```

---

## 4. 자주 하는 실수

| 증상 | 원인 | 해결 |
|------|------|------|
| 404 Not Found | Pages 빌드 아직 안 끝남 | 1~2분 대기 |
| `protected_domain_state: verified_domain_mismatch` | 타인이 같은 도메인을 선점 | 다른 서브도메인 사용 또는 GitHub 도메인 검증 |
| HTTPS 체크박스 회색(활성불가) | Let's Encrypt 발급 전 | 최대 24h 대기 후 재시도 |
| CNAME 끝 `.` 누락으로 실패 | DNS 문법 | 레코드 값 끝에 마침표 추가 |
| 루트 도메인에서 CNAME 사용 | CNAME + SOA 충돌 | Apex는 A 레코드 사용, 또는 ALIAS/ANAME 지원하는 DNS 사용 |

## 5. 삭제/변경

```powershell
# 도메인 해제

gh api -X DELETE repos/DongsooJung/stargate-ai-gallery/pages
```
