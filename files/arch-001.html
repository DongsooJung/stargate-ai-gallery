Built-in Tools

Claude의 손과 발

Claude Code에 내장된 기본 도구들입니다. 파일 읽기/쓰기, 명령어 실행, 검색 등 가장 근본적인 동작을 수행합니다. 모든 다른 개념의 기반이 됩니다.

Read Write Edit Bash Glob Grep Task WebSearch

주요 도구 목록

도구역할
Read파일 내용 읽기 (이미지, PDF 포함)
Write파일 생성/덮어쓰기
Edit기존 파일의 특정 부분 교체
Bash쉘 명령어 실행
Glob파일 패턴 검색 (예: *.ts)
Grep파일 내용 정규식 검색
Task서브에이전트 생성
WebFetch/Search웹 콘텐츠 가져오기/검색

핵심 포인트

항상 사용 가능하며, Settings의 권한 설정으로 허용/거부를 제어할 수 있습니다. MCP 도구도 이 도구들과 동일한 방식으로 사용됩니다.

Settings

모든 것의 제어판

JSON 설정 파일로 Claude Code의 권한, 환경 변수, Hooks, 플러그인 활성화, UI 언어 등 모든 동작을 제어합니다. 여러 범위(managed, user, project)에서 계층적으로 적용됩니다.

권한 제어 환경 변수 Hooks 정의 Plugin 활성화

설정 파일 우선순위 (높은순)

범위위치
Managed/Library/.../managed-settings.json
CLI 인수명령줄 플래그 (세션 한정)
로컬 프로젝트.claude/settings.local.json
공유 프로젝트.claude/settings.json
사용자~/.claude/settings.json

핵심 포인트

Hooks, Plugin 활성화, 권한 규칙 등이 모두 Settings 안에 정의됩니다. "설정의 설정"이자 모든 것의 컨테이너입니다.

CLAUDE.md

세션을 넘는 기억

매 세션 시작 시 자동으로 로드되는 마크다운 파일입니다. 프로젝트 구조, 코딩 규칙, 아키텍처 결정 등 영구적 지침을 담습니다. Skills와 달리 항상 컨텍스트에 포함됩니다.

항상 로드 영구 규칙 프로젝트 컨텍스트

파일 위치

범위위치
사용자 전역~/.claude/CLAUDE.md
프로젝트 공유CLAUDE.md 또는 .claude/CLAUDE.md
프로젝트 로컬.claude/CLAUDE.local.md

CLAUDE.md vs Skills

CLAUDE.md는 항상 로드 (컨텍스트 비용 고정), Skills는 필요할 때만 로드 (컨텍스트 절약). 보편적 규칙은 CLAUDE.md에, 특수 워크플로우는 Skills에 넣습니다.

Skills

온디맨드 전문가

필요할 때만 로드되는 도메인 전문 지식 패키지입니다. SKILL.md 파일에 지침을 정의하고, Claude가 관련성이 있으면 자동 호출하거나, 사용자가 /skill-name으로 수동 호출합니다.

SKILL.md 온디맨드 /slash-command $ARGUMENTS

SKILL.md 구성

필드설명
name스킬 이름 (슬래시 커맨드)
description자동 호출 판단 기준
context: fork별도 서브에이전트에서 실행
allowed-tools사용 가능 도구 제한
!`cmd`동적 컨텍스트 주입 (쉘 실행)

저장 위치

~/.claude/skills/ (개인), .claude/skills/ (프로젝트), 플러그인 내부 (네임스페이스). 동적 컨텍스트 주입 !`cmd`으로 라이브 데이터를 스킬에 넣을 수 있습니다.

Agents / Subagents

병렬 분신술

Task 도구로 생성되는 독립 AI 에이전트입니다. 각자 별도의 컨텍스트 윈도우에서 작업하며, 최대 10개까지 동시 실행됩니다. 무거운 작업을 메인 대화에서 분리합니다.

Task 도구 독립 컨텍스트 병렬 실행 커스텀 에이전트

내장 에이전트 타입

타입모델용도
ExploreHaiku빠른 코드베이스 탐색
Plan상속구현 계획 설계
general-purpose상속모든 도구 사용 가능
Bash상속터미널 명령 전용

핵심 제약

중첩 불가: 서브에이전트 안에서 또 다른 서브에이전트를 생성할 수 없습니다. 커스텀 에이전트는 ~/.claude/agents/에 마크다운 파일로 정의합니다.

Hooks

결정론적 자동화

특정 이벤트에서 항상, 확정적으로 실행되는 핸들러입니다. Claude의 판단 없이 조건이 맞으면 무조건 실행됩니다. 검증, 차단, 로깅, 알림 등에 사용합니다.

PreToolUse PostToolUse SessionStart Stop

주요 이벤트

이벤트시점차단 가능
SessionStart세션 시작✕
UserPromptSubmit사용자 입력 후✓
PreToolUse도구 실행 전✓
PostToolUse도구 실행 후✕
Stop응답 완료 시✓
PreCompact컨텍스트 압축 전✕

Hooks vs Skills

Hooks는 결정론적 (항상 실행), Skills는 확률적 (Claude가 판단). Hooks는 자동화/검증에, Skills는 전문 지식에 사용합니다.

MCP

외부 세계의 다리

Model Context Protocol - 외부 도구, 데이터베이스, API와 연결하는 표준 프로토콜입니다. MCP 서버가 도구를 제공하면, Claude는 내장 도구처럼 mcp__서버__도구 형식으로 사용합니다.

외부 API 연결 표준 프로토콜 mcp__server__tool

전송 방식

방식용도
http원격 클라우드 서비스 (권장)
sse서버 이벤트 (레거시)
stdio로컬 프로세스

설정 위치

~/.claude.json (사용자), .mcp.json (프로젝트), 플러그인 내부. 도구가 컨텍스트의 10% 이상을 차지하면 자동으로 Lazy Loading이 활성화됩니다.

Plugins

올인원 확장팩

Skills, Agents, Hooks, MCP, LSP를 하나로 묶은 배포 가능한 패키지입니다. 마켓플레이스에서 설치하며, 네임스페이스로 충돌 없이 관리됩니다. plugin.json 매니페스트로 정의합니다.

plugin.json 마켓플레이스 번들 패키지 버전 관리

플러그인에 포함 가능한 것

구성요소위치
Skillsskills/ (SKILL.md 파일)
Agentsagents/ (마크다운 파일)
Hookshooks/hooks.json
MCP 서버.mcp.json
LSP 서버.lsp.json (코드 인텔리전스)

핵심 포인트

단독 Skills/Hooks와 달리 버전 관리와 마켓플레이스 배포가 가능합니다. 네임스페이스: /plugin-name:skill-name

Built-in Tools

= 주방 도구

칼, 프라이팬, 오븐 등 요리의 기본 도구. 없으면 아무것도 만들 수 없습니다.

📋

Settings

= 주방 규칙

"가스레인지 사용 시 환풍기 켜기", "칼은 A 구역에서만 사용" 등 주방 운영 규칙.

📖

CLAUDE.md

= 레스토랑 매뉴얼

출근하면 항상 읽는 매뉴얼. "우리 가게는 한식 전문", "MSG 사용 금지" 같은 영구 규칙.

👨‍🍳

Skills

= 전문 레시피

특정 요리가 주문되면 꺼내 보는 상세 레시피. 항상 펼쳐두진 않지만 필요할 때 꺼냅니다.

👬

Subagents

= 보조 셰프

메인 셰프가 "디저트는 네가 해"라고 위임하면, 별도 조리대에서 독립적으로 작업합니다.

🚨

Hooks

= 자동 타이머 / 화재경보

오븐 열면 자동으로 타이머 시작, 연기 감지 시 무조건 경보. 의지와 무관하게 항상 작동합니다.

🚚

MCP

= 식재료 납품업체

외부에서 재료를 공급하는 업체. 표준 계약(프로토콜)으로 연결되어 다양한 식재료를 받을 수 있습니다.

🎁

Plugins

= 프랜차이즈 패키지

레시피 + 규칙 + 장비 + 납품업체를 한 번에 제공하는 프랜차이즈 시작 키트.
